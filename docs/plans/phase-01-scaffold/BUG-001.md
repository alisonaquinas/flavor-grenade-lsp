# Steps J–L unconditionally fail when test tier has no .test.ts files

> [!INFO] `BUG-001` · Bug · Phase 1 · Severity: `medium` · Priority: `medium` · Status: `open`

---

## Summary

`bun test tests/integration/`, `bun test tests/verification/`, and `bun test tests/validation/` each exit 1 with "no matching test files" when no `.test.ts` files exist in that directory. For scaffold and early phases these tiers are legitimately empty, but the phase-execution.md checklist calls `bun test <tier>/` unconditionally, causing Steps J–L to fail even when nothing is wrong.

---

## Reproduction Steps

1. Complete Phase 1 scaffold — no integration/verification/validation test files exist
2. Run `bun test tests/integration/`
3. Observe exit code 1 with message "The following filters did not match any test files"

---

## Expected Behaviour

A phase where a test tier has no test files should be able to complete steps J–L. "No test files in this tier" is not a test failure — it is a valid state for early phases.

---

## Actual Behaviour

`bun test tests/integration/` exits 1, blocking the phase from advancing.

---

## Violated Requirement

| Planguage Tag | Gist | Source File |
|---|---|---|
| — | Process correctness; `bun test` exit code semantics not accounted for in phase-execution.md | [[plans/phase-execution]] |

---

## Proposed Fix

Update `docs/plans/phase-execution.md` Steps J, K, and L to add a conditional: if a test tier directory contains no `.test.ts` or `.spec.ts` files, the step result is **N/A** (not a failure). The agent must note "no test files in this tier" in the phase retrospective. No code change required.
