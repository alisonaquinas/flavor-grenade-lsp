# Create extension directory and package.json manifest

> [!INFO] `TASK-137` · Task · Phase E1 · Parent: [[FEAT-015]] · Status: `done`

## Description

Create the `extension/` directory at the repository root and write `extension/package.json` with the full VS Code extension manifest. The manifest includes publisher identity (`alisonaquinas`), engine constraint (`^1.93.0`), activation events, all `contributes` sections (commands, configuration), workspace capabilities, esbuild-based build scripts, and runtime/dev dependencies. After writing the file, run `npm install` inside `extension/` to generate `package-lock.json` and populate `node_modules/`. This is the first task in the phase and has no dependencies.

---

## Implementation Notes

- Create `extension/` directory at the repo root

- Write `extension/package.json` with the exact content specified in the phase plan [[plans/phase-E1-extension-scaffold]]

- Key manifest fields: `publisher: "alisonaquinas"`, `engines.vscode: "^1.93.0"`, `extensionKind: ["workspace"]`, `main: "./dist/extension.js"`

- `vscode:prepublish` intentionally omits `build:server` — the server binary is cross-compiled separately (in CI by matrix job, locally by developer)

- `extensionKind: ["workspace"]` ensures the extension runs on the workspace host where vault files live

- Run `cd extension && npm install` after writing the file

- See also: [[plans/phase-E1-extension-scaffold]], [[adr/ADR015-platform-specific-vsix]]

## Notes

The `extension/` directory is a separate npm project from the root Bun project. It uses npm (not Bun) because VS Code extensions run in Node.js and `@vscode/vsce` expects npm-compatible `package-lock.json`. The `publisher` field must match the Marketplace-registered publisher ID exactly (no spaces).
